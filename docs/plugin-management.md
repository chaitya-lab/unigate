# Plugin Management System

## Overview

Plugins are discovered, registered, and managed through a unified system. Each plugin has a unique type-prefixed name for safe namespacing.

---

## Plugin Naming

### Format
```
{type}.{name}
```

Examples:
- `channel.telegram` - Telegram channel
- `channel.web` - Web webhook channel
- `match.text_contains` - Text contains matcher
- `match.sender` - Sender ID matcher
- `transform.truncate` - Truncate transform
- `transport.http` - HTTP transport

### Conflict Resolution
- Plugins with same name are not allowed
- System warns on duplicate registration
- User plugins override built-in with same name

---

## Plugin Registration

### Auto-discovery
Plugins are discovered from:
1. Built-in plugins (always loaded)
2. Configured plugin directories
3. Entry points (for pip-installed plugins)

### Manual Registration
```python
from unigate.plugins import register_plugin

def my_transform(msg, config):
    msg.text = f"[Prefix] {msg.text}"
    return msg

register_plugin("transform.my_prefix", my_transform)
```

### Discovery with CLI
```bash
# List all available plugins
unigate plugins list

# Show plugin details
unigate plugins show transform.truncate

# Generate config from plugins
unigate plugins gen-config > unigate.yaml
```

## Instance Configuration

Instances reference plugins by type-prefixed name:

```yaml
instances:
  my_telegram:
    type: channel.telegram
    config:
      token: !env:TELEGRAM_TOKEN
      mode: polling

  support_handler:
    type: handler

  sms_fallback:
    type: transport.http
    config:
      url: https://sms-gateway.example.com/send
```

### Type Resolution
The `type` field supports:
- Short name: `telegram` → `channel.telegram`
- Full name: `channel.telegram` → `channel.telegram`

## Routing Rules with Plugin References

Routing rules reference plugins:

```yaml
routing:
  rules:
    - name: urgent_to_sms
      match:
        from: telegram
        text_contains: URGENT
      transform:
        - transform.truncate_160
      forward_to:
        - transport.http
        - handler
```

### Missing Plugin Warning

If a referenced plugin is not found:
1. CLI warns on config load: `⚠️ Plugin 'transform.truncate_160' not found`
2. Rule with missing plugin is skipped (not registered)
3. Warning logged with rule name

## Plugin Activation

### Config-based
```yaml
plugins:
  enabled:
    - channel.telegram
    - channel.web
    - match.text_contains
    - transform.truncate
  
  disabled:
    - match.day_of_week  # Not needed
```

### Runtime Toggle (CLI)
```bash
# Disable a plugin
unigate plugins disable match.day_of_week

# Enable a plugin
unigate plugins enable match.day_of_week

# Reload all plugins
unigate plugins reload
```

## Plugin Status

### States
- **Available**: Discovered but not used
- **Enabled**: Registered and active
- **Disabled**: Registered but inactive
- **Error**: Failed to load

### Check Status
```bash
unigate plugins status

# Output:
# PLUGIN              TYPE       STATUS    CONFLICTS
# channel.telegram    channel    enabled   -
# channel.web        channel    enabled   -
# match.text_contains match      enabled   -
# transform.truncate transform   enabled   -
# transport.http     transport  disabled  -
```

## Config File Structure

```yaml
# unigate.yaml
unigate:
  plugin_dirs:
    - ./plugins           # Local plugins
    - ./custom_plugins    # Custom plugins

plugins:
  enabled:
    - channel.telegram
    - channel.web
    - match.text_contains
    - match.sender
    - match.has_media
    - transform.truncate
    - transform.extract_subject
    - transform.add_metadata
    - transport.http
  
  disabled: []

instances:
  telegram:
    type: channel.telegram
    enabled: true
    config:
      token: !env:TELEGRAM_TOKEN

  handler:
    type: handler
    enabled: true

routing:
  rules_file: ./config/routing/rules.yaml
  strict_mode: true  # Fail if plugin not found
```

## Implementation Notes

### Plugin Registry
```python
class PluginRegistry:
    _plugins: dict[str, PluginEntry]
    _enabled: set[str]
    
    def register(self, name: str, plugin: Any, source: str):
        if name in self._plugins:
            warn(f"Plugin '{name}' already registered from {self._plugins[name].source}")
            if source == "user":
                self._plugins[name] = PluginEntry(plugin, source)
        else:
            self._plugins[name] = PluginEntry(plugin, source)
    
    def get(self, name: str) -> Any | None:
        if name in self._enabled:
            return self._plugins[name].plugin
        return None
    
    def validate_rules(self, rules: list[RoutingRule]):
        for rule in rules:
            for transform in rule.actions.extensions:
                if not self.get(transform):
                    warn(f"Plugin '{transform}' not found for rule '{rule.name}'")
```

### Type Resolution
```python
def resolve_type(type_str: str) -> str:
    """Resolve short type to full type."""
    if "." in type_str:
        return type_str
    
    type_map = {
        "telegram": "channel.telegram",
        "web": "channel.web",
        "whatsapp": "channel.whatsapp",
        "handler": "handler",
        "transport.http": "transport.http",
    }
    return type_map.get(type_str, f"channel.{type_str}")
```
